# STM32 Nucleo F446RE — UART Hello World

> Your MCU just learned to talk. First words: `Hello from STM32!`

UART is how embedded systems speak to the outside world — debug logs,
sensor data, shell interfaces. If GPIO was waving hello,
UART is actually saying it.

---

## What is UART?

**Universal Asynchronous Receiver-Transmitter** — a serial protocol that sends
data bit by bit over two wires: TX (transmit) and RX (receive).

No clock line. No chip select. Just two wires and an agreed-upon speed
(baud rate). That simplicity is why UART shows up everywhere:

- Debug logging during development
- GPS, GSM, and Bluetooth modules
- Bootloaders (the STM32 factory bootloader is UART)
- Linux serial console on embedded Linux boards

---

## Hardware

Nucleo F446RE + USB cable. That's it — no extra hardware needed.

The Nucleo's onboard ST-Link acts as a **USB-to-UART bridge**, connecting
USART2 directly to your PC over the same USB cable you use to flash.

| Pin | Function |
|---|---|
| PA2 | USART2 TX |
| PA3 | USART2 RX |

> These pins are internally routed to the ST-Link — **nothing to wire.**

## CubeMX Configuration

**Peripheral:** USART2 → Mode: Asynchronous

| Parameter | Value |
|---|---|
| Baud Rate | 115200 bits/s |
| Word Length | 8 bits |
| Parity | None |
| Stop Bits | 1 |
| Flow Control | Disabled |
| Data Direction | Receive and Transmit |
| Oversampling | 16x |

> 8N1 at 115200 is the de facto default for embedded UART — 
> you'll use this config in almost every project.

## The Code

```c
char msg[] = "Hello from STM32!\r\n";

while (1)
{
    HAL_UART_Transmit(&huart2, (uint8_t*)msg, strlen(msg), HAL_MAX_DELAY);
    HAL_Delay(500);
}
```

| Part | What it does |
|---|---|
| `&huart2` | Handle for USART2, generated by CubeMX |
| `(uint8_t*)msg` | Cast string to byte array — UART sends raw bytes |
| `strlen(msg)` | Send exactly this many bytes, no more |
| `HAL_MAX_DELAY` | Block until transmission complete (polling mode) |

> `\r\n` — carriage return + newline. Without `\r`, 
> most terminal emulators won't move to the start of the next line.

## Seeing the Output

Open **Tera Term** (or any serial terminal):

- Port: whichever COM port your Nucleo appears as
- Speed: **115200**
- Data: 8 bit | Parity: None | Stop: 1 | Flow: None

Hit connect — you should see `Hello from STM32!` printing every 500ms.
